## Publish
1.  **Собрать пакет** `npm run build` 
3.  Запаковать 'npm pack`
4.  Удалить артефакты разработки `html360 uninstall-menu` `npm unlink` `npm uninstall -g html360`    
5.  Установить глобально из архива `npm install -g ./html360-x.x.x.tgz`
6.  **Проверить все команды html360**
7.  **Проверить CHANGELOG** Обратить вниание на дату релиза
8.  Создать коммит в мастер: `feat: release x.x.x`, `Update README.md`, ...
9.  **Создать коммит выпуска новой версии** `npm version x.x.x && npm run build && git push origin --follow-tags`
10. **Последний раз все проверить**
11. Создать релиз на github (опционально)
12. Логин `npm login`
13. Выложить `npm publish`

### В какое техническое ограничение уперся pannellum? 
Я пытался передавать pannellum оригинальную панораму 12K но не заметил разницы, когда передаешь в pannellum туже панораму но сжатую через sharp до 8K.

Это отличный вопрос, который вскрывает главную причину существования Multiresolution. Ты не заметил разницы, потому что уперся в лимиты видеокарты (GPU) и браузерного движка WebGL.
Вот три конкретных ограничения, из-за которых 12K превращается в «тыкву» без нарезки на тайлы:
1. Лимит на размер текстуры (MAX_TEXTURE_SIZE)
У каждой видеокарты есть жесткий предел на размер одной картинки, которую она может переварить за раз.

* На большинстве современных ПК это 16384px (16K).
* На многих смартфонах и старых ноутбуках — всего 8192px (8K) или даже 4096px.
* Что происходит: Если ты скармливаешь Pannellum файл 12K на устройстве с лимитом 8K, браузер либо принудительно сжимает картинку до 8K (вот почему разницы нет!), либо просто выдает черный экран/ошибку.

2. Память GPU (VRAM)
Когда ты открываешь обычную панораму (один файл), видеокарта должна распаковать её целиком и держать в видеопамяти.

* Панорама 12K (12288 x 6144) в несжатом виде занимает: $12288 \times 6144 \times 4$ байта (RGBA) $\approx$ 300 МБ.
* Для мобильных браузеров это огромный кусок. Чтобы вкладка не вылетела, браузер часто «мылит» изображение, чтобы сэкономить память.
* Sharp vs Браузер: Sharp сжимает умнее. Поэтому 8K от Sharp может выглядеть четче, чем 12K, которую браузер «задушил» своими фильтрами при нехватке памяти.

3. Отсутствие Mip-mapping (Детализация при зуме)
В режиме одного файла Pannellum рисует всю сферу сразу. Когда ты начинаешь «зумить» (приближать):

* Ты пытаешься рассмотреть 100 пикселей исходника на 100 пикселях экрана.
* Но так как текстура одна и она огромная, возникают артефакты интерполяции (дрожание или размытость).
* Multiresolution решает это: когда ты зумишь, он подменяет кусок сферы на новый, очень четкий тайл, который изначально был снят в 16K, но весит мало, так как это маленький квадратик.

Почему Multires — это Pro-уровень?
С нарезкой на тайлы:

   1. Лимит GPU не важен: Мы показываем много маленьких тайлов по 512px.
   2. Экономия памяти: В памяти только те 10-15 тайлов, которые ты видишь прямо сейчас (~10-20 МБ вместо 300 МБ).
   3. Честные 16K/32K: Ты реально видишь каждую травинку при максимальном приближении, потому что смотришь на фрагмент оригинального разрешения без сжатия браузером.

Итог: 12K «одним куском» — это лотерея (у кого-то сработает, у кого-то нет). 12K через Multires — это гарантированное качество на любом устройстве.

### Как узнать MAX_TEXTURE_SIZE
```
(function checkWebGL() {
    const canvas = document.createElement('canvas');
    const gl = canvas.getContext('webgl') || canvas.getContext('experimental-webgl');

    if (!gl) {
        console.error("WebGL не поддерживается в этом браузере!");
        return;
    }

    const maxTextureSize = gl.getParameter(gl.MAX_TEXTURE_SIZE);
    const vendor = gl.getParameter(gl.UNMASKED_VENDOR_WEBGL || gl.VENDOR);
    const renderer = gl.getParameter(gl.UNMASKED_RENDERER_WEBGL || gl.RENDERER);

    console.log("%c--- WebGL Hardware Report ---", "color: #00ff00; font-weight: bold;");
    console.log(`GPU: ${renderer} (${vendor})`);
    console.log(`Лимит текстуры (Max Texture Size): %c${maxTextureSize}px`, "color: #ff9900; font-weight: bold;");
    
    if (maxTextureSize < 12288) {
        console.warn("⚠️ Твоя панорама 12K (12288px) будет принудительно сжата браузером!");
    } else {
        console.log("✅ Твоё железо тянет 12K целиком.");
    }
    console.log("------------------------------");
})();
```
